# Property Lens — v0 Internal Demo

The operating window into DMG's structured photo evidence layer. Pick a property, see its 10 standardized photo plots on a map, click any plot to see the provider photo alongside the slot specification, and inspect capture-rate analytics across the pilot portfolio.

## Quick Start

```bash
npm install
npm run build-data    # parse KMLs → data/properties.json + copy photos
npm run dev           # dev server at http://localhost:3000
```

## What It Does

| Surface | Route | Description |
|---|---|---|
| Property Library | `/` | Searchable list of all pilot properties + multi-property map |
| Property Detail | `/property/[id]` | Parcel polygon, 10 numbered pins, diagram overlay, slot list |
| Photo Point Card | modal on detail view | Provider photo + slot spec side by side, Hawk-Eye verdict |
| Analytics | `/analytics` | Capture rate by slot, property scoreboard, slot × property heatmap |

Deep-link to a specific slot: `/property/03_pavilion_newport?slot=PP-03`

## Adding a New Property

1. Run the `enrich-photo-plots` skill (in DMG Data MCP) for the new site. It produces:
   - `NN_property_name/` folder with `*-enriched.kml`, `*-parcel-boundary.kml`, `ADDRESS.txt`

2. Drop the folder into `/Users/brian.back/photo-plots/NN_*`

3. Run `npm run build-data` — the script auto-discovers the folder, parses the KML, and copies any photos into `public/photos/`

4. Restart dev or rebuild

## What Is Static vs Dynamic

| Thing | Status in v0 |
|---|---|
| Property metadata (names, coordinates, slot specs) | Static JSON built from KML at build time |
| Provider photos for Newport Pavilion | Static files in `public/photos/03_pavilion_newport/` |
| Hawk-Eye verdicts (Newport) | Hardcoded mock in `scripts/build-data.ts` |
| Analytics widget numbers | Hardcoded mock in `components/AnalyticsClient.tsx` |
| Map tiles | Live from OpenStreetMap (free, no key required) |
| Other properties (no photos) | Slot pins show as "Missing"; slot spec still renders |

## Tech Stack

- **Next.js 16** (App Router) — TypeScript strict mode
- **Tailwind CSS v4** — utility-first styling
- **MapLibre GL JS** — open-source map renderer, no API key required
- **OSM tiles** — `tile.openstreetmap.org` for streets, Esri World Imagery for satellite
- **Recharts** — analytics widgets
- **fast-xml-parser** — KML parsing in build script

> **Mapbox note:** The PRD specifies Mapbox GL JS. v0 uses MapLibre GL JS (API-compatible fork) to avoid requiring a token for demos. For v1 production, swap `MAP_STYLE_STREETS`/`MAP_STYLE_SATELLITE` in `components/PropertyMap.tsx` for Mapbox style URLs and add `NEXT_PUBLIC_MAPBOX_TOKEN` to the environment.

## Project Layout

```
app/
  page.tsx                   # Property Library (home)
  property/[id]/page.tsx     # Property Detail View
  analytics/page.tsx         # Analytics Dashboard
  layout.tsx
components/
  PropertyMap.tsx            # MapLibre wrapper — library + detail modes
  PropertyLibraryClient.tsx  # Search, filter, list
  PropertyDetailClient.tsx   # Detail layout + diagram toggle
  PhotoPointCard.tsx         # Photo + slot spec modal
  SlotList.tsx               # Slot side panel
  AnalyticsClient.tsx        # Three analytics widgets + summary stats
  TopNav.tsx
lib/
  types.ts                   # Property, PhotoPoint, Capture types
  data.ts                    # Load properties.json at build time
data/
  properties.json            # Generated by build-data
public/photos/               # Provider photos (copied by build-data)
scripts/
  build-data.ts              # KML → properties.json pipeline
```
